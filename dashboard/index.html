Complete beginner setup — step by step
Step 1 · 5 minutes
Create your GitHub account (if you don't have one)
Go to github.com → Sign up → Create a new repository called baby-sleep-agent → Set it to Private.
Step 2 · 5 minutes
Create your Apify account — no credit card needed to start
Go to apify.com → click Continue with GitHub → one click, done. You get $5 free credits monthly — enough for all weekly runs. Find your API token at: apify.com/account → Integrations → Personal API tokens → copy it.
Step 3 · 10 minutes
Get your Anthropic API key
Go to console.anthropic.com → API Keys → Create Key → copy it. Add $5–10 credits — weekly runs cost roughly $0.20–0.50 each.
Step 4 · 5 minutes
Upload the agent code to your GitHub repo
Download the code package (all files) → drag and drop them into your GitHub repo using the web UI. No terminal needed — GitHub has an "Upload files" button on the repo homepage.
Step 5 · 5 minutes
Add your API keys as GitHub Secrets
In your repo → Settings → Secrets and variables → Actions → New repository secret. Add two secrets:
APIFY_API_TOKEN = your Apify token
ANTHROPIC_API_KEY = your Anthropic key
These are encrypted — no one can see them, not even you after saving.
Step 6 · Done!
Test the agent runs correctly
In your repo → Actions tab → "Baby Sleep Content Agent" → Run workflow → click green Run workflow button. Watch it run live. Check the Logs tab in this dashboard after. Every Monday at 6am SGT it will run automatically from now on.
Your .env file (for local testing only)
APIFY_API_TOKEN=apify_api_xxxxxxxxxxxx ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
Never commit this file to GitHub. The .gitignore already excludes it.

Glossary — what these terms mean
✅ Complete run
All 4 Gemini AI calls succeeded. Your blog template, Instagram carousel, reel script, and gap analysis were all generated with real AI content this week.
⚠️ Fallback
One or more Gemini calls hit an error even after all retries. That section shows a placeholder message starting with "[…unavailable…]" instead of real content. The rest of the report is fine.
429 — Rate limited
Gemini received too many requests in a short window. Most common cause. Usually clears within 1 hour. Safe to re-run the agent.
503 / 500 — Server error
Google's Gemini servers had a temporary problem. Nothing you did wrong. Re-running the agent after 30 minutes almost always fixes it.
Fallback rate
The percentage of runs where at least one section used a placeholder. Under 20% is normal. Over 20% consistently means your Gemini quota needs attention.
